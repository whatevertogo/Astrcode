# Codebase Exploration

Start with directory and search tools to narrow scope, then read the specific files that explain behavior. Avoid guessing from filenames alone.

## Workflow

1. **Narrow scope first** — use directory listings and search tools to locate relevant areas.
2. **Read specific files** — understand actual implementations rather than inferring from names.
3. **Avoid assumptions** — never make architectural claims without reading the code that backs them.
